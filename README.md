## Overview

Given a set of patient attributes (age, sex, blood pressure, cholesterol level, and sodium-to-potassium ratio), the model predicts which of five drug classes (`DrugY`, `drugC`, `drugX`, `drugA`, `drugB`) is appropriate for that patient.

The full workflow is:

1. Load and shuffle the dataset.
2. Preprocess features (encoding + imputation + scaling) inside a `scikit-learn` `Pipeline`.
3. Train a `RandomForestClassifier`.
4. Evaluate with accuracy and macro-F1 score, then save a confusion-matrix plot.
5. Persist the trained pipeline with `joblib`.
6. Serve predictions through a Gradio UI deployed on Hugging Face Spaces.

## Dataset

**File:** `data/drug200.csv`  
**Source:** UCI Drug Classification dataset (200 samples, 6 columns)

| Column        | Type        | Description                                      |
|---------------|-------------|--------------------------------------------------|
| `Age`         | Numerical   | Patient age (15 – 74)                            |
| `Sex`         | Categorical | `M` (Male) or `F` (Female)                       |
| `BP`          | Categorical | Blood pressure level: `HIGH`, `LOW`, `NORMAL`    |
| `Cholesterol` | Categorical | Cholesterol level: `HIGH` or `NORMAL`            |
| `Na_to_K`     | Numerical   | Sodium-to-potassium ratio in blood (6.2 – 38.2)  |
| `Drug`        | Target      | Drug class: `DrugY`, `drugC`, `drugX`, `drugA`, `drugB` |

The dataset is **shuffled** with `random_state=42` before splitting.

## Machine Learning Pipeline

All steps are encapsulated in a single `scikit-learn` `Pipeline` so that the same transformations applied at training time are automatically applied at inference time.

### Preprocessing

A `ColumnTransformer` handles the two types of input features in parallel:

| Transformer      | Columns (indices) | Feature Names              | Description                                      |
|------------------|-------------------|----------------------------|--------------------------------------------------|
| `OrdinalEncoder` | 1, 2, 3           | `Sex`, `BP`, `Cholesterol` | Encodes categorical strings to integer ordinals  |
| `SimpleImputer`  | 0, 4              | `Age`, `Na_to_K`           | Fills missing numerical values with the median   |
| `StandardScaler` | 0, 4              | `Age`, `Na_to_K`           | Scales numerical features to zero mean / unit variance |

### Model

- **Algorithm:** `RandomForestClassifier` (scikit-learn)
- **Estimators:** 100 decision trees (`n_estimators=100`)
- **Random seed:** `125` (for reproducibility)
- **Train / Test split:** 70% train, 30% test (`test_size=0.3`, `random_state=125`)

### Evaluation

After training, the model is evaluated on the held-out test set:

| Metric       | Value  |
|--------------|--------|
| **Accuracy** | 97%    |
| **F1 Score** | 0.94 (macro average) |

Metrics are saved to `results/metrics.txt` and a confusion matrix is saved to `results/model_results.png`.

## Gradio Web Application

**File:** `app/app.py`

The app loads the saved `drug_pipeline.joblib` and exposes a web UI powered by [Gradio](https://gradio.app/).

### Inputs

| Input                        | Widget  | Range / Options                  |
|------------------------------|---------|----------------------------------|
| Age                          | Slider  | 15 – 74 (step 1)                 |
| Sex                          | Radio   | `M`, `F`                         |
| Blood Pressure               | Radio   | `HIGH`, `LOW`, `NORMAL`          |
| Cholesterol                  | Radio   | `HIGH`, `NORMAL`                 |
| Sodium to Potassium Ratio    | Slider  | 6.2 – 38.2 (step 0.1)           |

### Output

- **Predicted Drug** – the top predicted drug class label displayed as a `gr.Label`.

### Example Inputs

|-----|-----|----------------|-------------|---------|-----------------|

| 61  | F   | LOW            | HIGH        | 10.043  | drugC           |

The app is deployed on **Hugging Face Spaces** at `arjun122/mlcloud` using the `gradio` SDK.

## CI/CD Workflow

The `Makefile` defines all automation steps, which can be chained together to go from raw data to a live deployed model.

### Makefile Targets

| Target          | Command          | Description                                                                                     |
|-----------------|------------------|-------------------------------------------------------------------------------------------------|
| `install`       | `make install`   | Upgrades `pip` and installs all dependencies from `requirements.txt`                            |
| `train`         | `make train`     | Runs `train.py` to train the model and save outputs to `model/` and `results/`                  |
| `eval`          | `make eval`      | Builds a markdown report from `results/metrics.txt` and the confusion matrix, then posts it as a CI comment using [CML](https://cml.dev/) |
| `update-branch` | `make update-branch` | Commits the trained model and results to the `update` branch and force-pushes to GitHub      |
| `hf-login`      | `make hf-login`  | Authenticates with Hugging Face using `HF_TOKEN` environment variable                           |
| `push-hub`      | `make push-hub`  | Uploads `app/`, `model/`, and `results/` directories to the Hugging Face Space `arjun122/mlcloud` |
| `deploy`        | `make deploy`    | Runs `hf-login` followed by `push-hub` – full deployment to Hugging Face Spaces                 |

A typical end-to-end run from a clean state:

```bash
make install      # Install dependencies
make train        # Train and evaluate the model
make eval         # Post metrics as a CI comment (requires CML)
make deploy       # Deploy the app to Hugging Face Spaces (requires HF_TOKEN)
```

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/arjunv122/MLMODEL.git
cd MLMODEL
```

### 2. Install dependencies

```bash
make install
# or
pip install --upgrade pip && pip install -r requirements.txt
```

### 3. Train the model

```bash
make train
# or
python3 train.py
```

This will:
- Load `data/drug200.csv`
- Train the Random Forest pipeline
- Print accuracy and F1 score to the console
- Save `results/metrics.txt` and `results/model_results.png`
- Save `model/drug_pipeline.joblib`

### 4. Run the Gradio app locally

```bash
cd app
python app.py
```

Open the URL shown in the terminal (typically `http://127.0.0.1:7860`) to interact with the prediction UI.

### 5. Deploy to Hugging Face Spaces

```bash
export HF_TOKEN=<your_huggingface_token>
make deploy
```
